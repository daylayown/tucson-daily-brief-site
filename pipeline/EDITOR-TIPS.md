# Editor Tips — Manual Story Injection

Stories the editor (Nicholas) submitted by hand for the daily brief — leads that may
not appear in any sources.json feed yet. The 6 AM briefing agent reads this file every
run (see the "Editor Tips" section of TUCSON-BRIEF.md).

How it works:
- Include any tip whose **include-through** date is today or later, ranked by the normal
  Editorial Priorities. Skip (do not include) tips whose include-through date has passed.
- You MAY fetch the source link(s) in a tip — an explicit exception to the sources.json-only
  rule, because the editor has vetted them.
- Verify against the link(s) and any newer coverage. Attribute to the outlet(s). If a story is
  single-source or not yet widely confirmed, soft-hedge ("is scheduled to," "according to
  [outlet]") and flag it as single-source.
- Do not invent details beyond the tip and its linked source(s).
