### Local names and places bible

Built 2026-05-12 during an editorial review of a Pima County BOS draft that surfaced systematic Deepgram errors (Cano → Conner, Cullen → Cohen, Cuaron → Quaron, Acuña → Cunno, Heinz → Hines, Winfield → Rainfield/Wynne/Ranfeld). The bible is a single JSON file holding canonical names, titles, and known mistranscriptions for every regularly-cited official across the four municipalities plus regional/state players. Loaded by `ai_reporter.py` and injected into the report-generation system prompt so Sonnet writes the correct name on first reference and fixes pronouns to match.

**File:** `pipeline/local_names.json`. Top-level keys: `pima-county`, `tucson`, `marana`, `orovalley` (per-jurisdiction rosters) and `regional` (countywide elected officials, state-level players like the AZ Auditor General, regional landmarks). Each bucket has `people[]` and `places[]` arrays with `canonical`, `title`, optional `pronouns`, and `deepgram_misreads` arrays. A `_meta.verification_needed` array flags entries that couldn't be fully confirmed (Suzanne Droubie, Dan Hunt; plus, as of 2026-07-14, the staff whose pronouns are still unset and Michael Racy's contract status) so they get re-checked before they appear in a real transcript.

**Integration in `ai_reporter.py`:** `municipality_from_slug()` matches the transcript slug prefix to a municipality, and `load_local_names_reference()` formats the relevant municipality plus the shared `regional` bucket into a "REFERENCE — CANONICAL NAMES AND PLACES" block injected after the speaker note. Unknown slugs return empty cleanly. Per-call overhead: ~500-700 tokens (fractions of a cent).

**How to grow the bible:** every editorial review yields new misreads — add them to the relevant entry's `deepgram_misreads` array as you encounter them. New officials surfacing in a meeting (e.g., the Pima County Elections Director, AZ Auditor General team) should be added to the appropriate bucket. Bible is the single source of truth; no separate post-pass regex layer is needed at this scale.

**Pronouns are load-bearing (added 2026-07-14).** The `pronouns` field existed in the schema but was unpopulated for every Pima County entry, so the 2026-07-14 BOS draft misgendered Supervisor Andrés Cano ("she wanted clearer information") — the model had nothing to anchor to and inferred from the name. Pronouns are now populated for the evidenced entries and `load_local_names_reference()` renders them, but the fix is the *prompt*: it now says to use the given pronouns exactly, and — for anyone **not** in the bible — to never infer pronouns from a name (write around it or use they/them). **Only add pronouns you can evidence** — a transcript honorific/pronoun ("miss Lesher", "his audio cut out", "since he's the"), or documented public record. If unknown, leave the field off and add the person to `_meta.verification_needed` rather than guessing; a wrong guess misgenders a real person in a way the neutral default never does.

**The bible can launder a misread into "canonical" — and then certify it (found + fixed 2026-07-16, commit `e31d836`).** The entry for the Tohono O'odham Nation's chairman read `"canonical": "Verlan Jose"`. That is not his name — it's **what Deepgram heard**. It is **Verlon M. Jose**, verified against [the Nation's tribal government page](https://www.tonation-nsn.gov/tribal-government/) and a House Appropriations bio. The entry had been created *from* a transcript by copying the misread and stamping it canonical, with no external check — so the system that exists to catch Deepgram name errors instead **taught the model the error and blessed it**, and the 7/14 BOS report published a sitting tribal chairman's name misspelled. Fixed: canonical is now `Verlon Jose`, with `"Verlan Jose"`/`"Verlan"` recorded as misreads so the transcript form gets corrected on sight.

**The detectable signature: an entry with `deepgram_misreads` absent or empty.** A *verified* entry accumulates misreads (Stahl Hamilton has three, Zraick has three) because real correction leaves a trail. An entry with none either never appeared in a transcript, or — the dangerous case — **it entered by transcription and there was nothing to correct it against**. That absence is the audit key.

**Known structural gap: the reporter is deaf to items that pass without discussion (found 2026-07-14).** The AI reporter drafts from the transcript, so anything approved *silently* is invisible to it — bundled items, the consent calendar, "as recommended by staff" motions. On 2026-07-14 the single highest-reader-impact decision of the night (item 26: Outside Agency funding cut from $10,059,380 requested to $3,936,620 — ~61%, hitting the Community Food Bank, Emerge's domestic violence services, homeless case management) passed **4-1 with not one word said about its substance**, and was absent from the published draft until caught by hand. This is the cruel irony of the pipeline: **items that pass without debate are exactly the ones most worth covering** — it's the entire premise of "Buried in the Agenda" — and the news reporter is blind to them by construction. **The fix already exists in-house:** `agenda_mining.py` had flagged this item as the *lead* of its 2026-07-09 preview, and the Legistar packet carries each item's full text + agenda number (`EventItemAgendaNumber`, note the trailing period — `"26."`). Cross-referencing the agenda item list against what the transcript actually discusses would surface "voted on, never debated" items automatically. **Not built — gated per [[feedback_resist_feature_creep]]; capture only.** Until then, when reviewing any Pima draft, diff the Legistar agenda against the draft's coverage by hand.

**Editorial VERIFY markers + publish guard (added 2026-07-14).** Drafts can carry inline `VERIFY:` notes flagging claims the transcript doesn't support (e.g. a 4-1 vote where the chair never names the dissenter). These are **not** safe as bare HTML comments: `report_md_to_html` escapes HTML, so a leftover marker renders as *literal visible text* on the live page rather than hiding in the source. So `ai_reporter.py` now **hard-stops** on `--approve`/`--publish` if any `VERIFY:` marker remains, printing each one as a checklist (escape hatch: `--allow-unverified`, same idiom as the newsletter's `--allow-missing-puzzle`). Resolve the note and delete the marker; don't route around the guard.

**First production proof:** the 2026-05-12 Pima County BOS regular meeting draft (Nanos perjury referral). Every Pima County name came through correctly on first reference — Cano with accent, Heinz not Hines, Cázares-Kelly with accent, DeBonis spelled right — and editorial review focused entirely on substantive issues (vote tally, attribution, framing) rather than name corrections.
